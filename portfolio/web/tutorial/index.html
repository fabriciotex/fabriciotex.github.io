What is regex?

Regex is short for Regular Expressions, that are expressions that deal with string inputs and perform a comparison and check if the string match specified rules. In JavaScript Regular expressions are also objects that contains a number of methods, returning diverse operation results.

Regular expressions are usually applied on input validations to ensure that the user is typing a string according to the script rules, ensuring its functionlaity, for example, a script might need a telephone number in the format (###) ### ####, and so, the regular expression is used to compare the input string to this rule and therefore validates it.

Besides JavaScript there are some other languages that accept the use of regex. The JavaScript regex is based on the Perl language use.

Pattern

A regex can be defined in two ways: 

                        
                            // using a shorthand notation
                            var regex = /abc/;

                            // or using the object constructor
                            var regex = new RegExp("abc");

Note that the pattern is only enclosed in quotation marks when using the constructor version.

You can specify a modifier to make your regular expression ignore case sensitivity, for example. 

                        
                            // looking for only uppercase ABC sequence
                            var regex = /ABC/;

                            // looking for any abc sequence
                            var regex = /ABC/i;

You can also send a RegEx as parameter in methods, using the shorthand notation: 

                        
                            var search = string.search(/abc/);

A RegEx can be composed by simple or special characters. The examples above consider only simple characters, that will be searched on the string like they are shown, in that specific order containing exactly those 3 letters, being one a, one b and one c, we also have to remember that capitalization matters, unless you explicitly tells using the "i" modifier. For more complex searches we should understand consider the use of the special characters on the tables below:

Table of Modifiers
Character Description
g Looks for all matches, not stopping at the first found.
m Perform a multiline search.
i Ignore case sensitivity
Table of Metacharacters
Character Description Example
. Matches any single character (except the new line character). "an", "on", "en", "?n".
\ Identify the next character according to the following rules: If the next character is a non-special is identified as a special character, not being interpreted literally. If a special character follows the backslash the character is interpreted as literal by the regular expression, also known as escaping the character. \n is interpreted as a new line character.
\' is interpreted as the single quotation mark '.
^ Matches the beginning of the input string. In "An endless story", the expression /^A/ returns true.
However in "an Adventure" returns false.
$ Matches the end of the input string. In "An endless story", the expression /y$/ returns true.
However in "cozy home" returns false.
\w Matches a word character. In "Discount = 100%", the expression /\w/ matches "Discount100"
\W Matches a non-word character. In "Discount = 100%", the expression /\w/ matches " = %"
\d Matches a digit character. In "Discount = 100%", the expression /\w/ matches "100"
\D Matches a non-digit character. In "Discount = 100%", the expression /\w/ matches "Discount = %"
\s Matches a space character. In "Discount = 100%", the expression /\w/ matches " " (two space characters)
\S Matches a non-space character. In "Discount = 100%", the expression /\w/ matches "Discount=100%"
\b Matches the character at the beginning or ending of a string. In "discount = hundred", the expression /\bd/g matches the "d" starting and ending the string.
\B Matches a non-space character. In "counting discounts = hundred", the expression /\Bcount/g matches the "count" in the middle of the word "discount".
\n Matches a new line character. In the string "Lorem ispum dolor\nsit amet" the expression /\w\n/ will match "Lorem ispum dolor"
Table of Quantifiers
Character Description Example
* Matches 0 or more occurrences of a character (equivalent to {0,}). The expression /ho*/ returns true in "Yahoo", "house", and in "hiccup".
+ Matches 1 or more occurrences of a character (equivalent to {1,}). The expression /ho+/ returns true in "Yahoo" and "house", and false in "hiccup".
? Matches the preceding expression 0 or 1 times, as an optional (equivalent to {0,1}). The expression /h?o+/ returns true in "Yahoo" and "Yaoo".
{x} Matches the preceding expression x or more times. The expression /ho{0}/ returns true in "Yahoo", "Yaho", and "Yeah".
{x,y} Matches the preceding expression x to y times. The expression /ho{1,2}/ returns true in "Yahoo" and "Yaho", and false in "Yeah".
{x,} Matches the preceding expression at least x times. The expression /ho{2,}/ returns true in "Yahoo" and false in "Yaho".
?= Matches the any string followed by the specific string after the quantifier. The expression /Ya?=hoo/ returns true in "Yahoo" and "Boohoo".
?! Matches the any string not followed by the specific string after the quantifier. The expression /Ya?!hoo/ returns false in "Yahoo" and "Boohoo" and true in "Yale" and "Boolean".
Table of Brackets
Character Description Example
[xyz] Matches all occurrences of any character between brackets In the string "There it is" the expression /[he]/g matches all "h" and "e" ocurrences.
[^xyz] Matches all occurrences that is not any character between brackets In the string "There it is" the expression /[^he]/g matches all characters that are not "h" and "e".
[0-9] Matches a range of characters from 0 to 9 In the string "1234567abc" the expression /[1-4]/ matches only "1234".
[^0-9] Matches all characters not in this range 0 to 9 In the string "1234567abc" the expression /[1-4]/ matches only "567abc".
(x|y) Matches one provided alternative or another. In the string "abc123def" the expression /(3|d)/ matches only the number 3, because it comes first on the order.
(xy) Matches the required characters. In the string "abc123def" the expression /(3d)/ matches the "3d", however, if the expression was /4g/ it would return false because there is no occurrence of it on the string.

Methods

For dealing with regular expressions we can rely on two main methods for this specific object, that can work together with some methods from the String Object.

Table of Methods
Methods Description
exec() Test for a match on a string returning true or false. 
                                    
                                        var regex = /owl/;
                                        var str = "The wolf howls";
                                        regex.exec(str); // returns "owl"
test() Test for a match on a string returning true or false. 
                                    
                                        var regex = /owl/;
                                        var str = "The wolf howls";
                                        regex.exec(str); // returns true

Examples

Here are a few examples on using regular expressions in a contact form. 

                            
                                function testName() {
                                    /* this function test a name input in the following way:
                                     * ^        anchors the beggining of string
                                     * ()       everything that is between parenthesis is required
                                     * ?:       the group is not captured (saved in memory) and therefore
                                                there's no need to appear again in the expression
                                     * \w+      has to contains one or more word characters
                                     * [.\']?   matches any character (.) followed by a literal quote character (')
                                                the '?' says that is optional for this to occurr
                                     *  ?       matches a space character 0 or more times
                                     * {1,7}    has to contain the whole expression from 1 to 7 times
                                     *          (1 to 7 names, separated by a space char)
                                     * $        anchors the end of the string
                                     */
                                    var regex = /^(?:\w+[.\']? ?){1,7}$/;
                                    var name = document.getElementById("name").value;
                                    var returnVal = document.getElementById("nameReturn");
                                    returnVal.innerHTML = "Returns: " + regex.test(name);
                                }
                            
                        

                            
                                function testEmail() {
                                    /* this function test an email input in the following way:
                                     * ^                anchors the beggining of string
                                     * [_\w\-]+         must have word characters or "_" (underscore character)
                                                        or "-" (dash) one or more times
                                     * (\.[_\w\-]+)*    should to contain any character followed by any of those
                                                        described above 0 or more times
                                     * @                must have an "@" character
                                     * [_\w\-]+         must have word characters or "_" (underscore character)
                                                        or "-" (dash) one or more times
                                     * (\.[_\w\-]+)*    should to contain any character followed by any of those
                                                        described above 0 or more times
                                     * (\.[\D]{2,6})    must have a "." character followed by a non-digit
                                                        from 2 to 6 times
                                     * $                anchors the end of the string
                                     */
                                    var regex = /^[_\w\-]+(\.[_\w\-]+)*@[\w\-]+(\.[\w\-]+)*(\.[\D]{2,6})$/;
                                    var email = document.getElementById("email").value;
                                    var returnVal = document.getElementById("emailReturn");
                                    returnVal.innerHTML = "Returns: " + regex.test(email);
                                }
